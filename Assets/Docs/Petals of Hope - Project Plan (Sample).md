# **Petals of Hope \- Project Plan / Production Schedule (Sample)**

## **1\. Introduction**

This document provides a high-level project plan and estimated production schedule for *Petals of Hope*. It outlines key phases, milestones, and potential timelines.

**Disclaimer:** This is a sample schedule based *only* on the provided design documents. Actual timelines depend heavily on team size, experience, budget, and unforeseen challenges. It assumes a small, dedicated indie team (e.g., 3-5 people: programmer, artist, designer/level builder, sound designer potentially part-time/contract).

## **2\. Project Goals**

* Develop a 2D pixel art platformer, *Petals of Hope*, based on the GDD.  
* Implement core mechanics: movement, jumping, combat, 3 special abilities (Double Jump, Wall Jump, Dash).  
* Create 40 levels across 4 distinct environments, including 4 boss fights.  
* Deliver on PC (Primary) with potential for Mobile adaptation (Secondary).  
* Achieve target performance (60 FPS) and visual style (pixel perfect).

## **3\. Key Phases & Estimated Timeline**

*(Timeline is indicative and assumes full-time development)*

### **Phase 1: Pre-Production (4-6 Weeks)**

* **Goal:** Finalize design, establish workflow, prototype core mechanics.  
* **Tasks:**  
  * Refine GDD, LDD, Art Style, Sound Design documents.  
  * Set up Unity project, version control (Git), task management.  
  * Prototype core player controller (movement, jump, basic physics).  
  * Develop initial art assets for Elias and Forest environment (tileset, simple enemy).  
  * Prototype basic enemy AI (patrolling wolf).  
  * Establish sound effect workflow.  
  * Technical feasibility tests (pixel perfect rendering, mobile performance if applicable early).  
* **Milestone:** Playable prototype demonstrating core movement and feel in a test environment. Finalized core design documents.

### **Phase 2: Production \- Vertical Slice (8-12 Weeks)**

* **Goal:** Create a polished "vertical slice" showcasing core gameplay loop and quality target. Typically includes 1-2 full levels from the first environment.  
* **Tasks:**  
  * Develop Levels 1-3 (Forest) to near-final quality.  
  * Implement Forest enemies (Wolf, Spider, Archer Elf) with full AI.  
  * Implement collectibles (Coins, Health) and basic HUD.  
  * Create finalized art assets for Elias and Forest environment.  
  * Implement Forest ambient sounds and basic SFX.  
  * Develop checkpoint system.  
  * Basic Main Menu and Pause Menu functionality.  
* **Milestone:** Polished Vertical Slice (e.g., Levels 1-2) demonstrating gameplay, art, sound, and UI. Ready for initial playtesting/feedback.

### **Phase 3: Production \- Content Creation (24-36 Weeks)**

* **Goal:** Develop all remaining levels, enemies, bosses, and abilities.  
* **Tasks (Iterative per Environment):**  
  * **Forest Completion (Levels 4-10 \+ Boss):**  
    * Build remaining Forest levels.  
    * Implement Forest Boss mechanics and art.  
    * Implement Double Jump ability and associated level design challenges.  
    * Finalize Forest art and sound assets.  
  * **Desert Development (Levels 11-20 \+ Boss):**  
    * Create Desert art assets (tilesets, enemies, boss).  
    * Build Desert levels, implementing hazards (quicksand, etc.).  
    * Implement Desert enemies and Boss.  
    * Implement Wall Grab/Jump ability.  
    * Integrate Desert music and SFX.  
  * **Mountain Development (Levels 21-30 \+ Boss):**  
    * Create Mountain art assets.  
    * Build Mountain levels, implementing hazards (ice, wind).  
    * Implement Mountain enemies and Boss.  
    * Implement Dash ability.  
    * Integrate Mountain music and SFX.  
  * **Cave Development (Levels 31-40 \+ Final Boss):**  
    * Create Cave art assets.  
    * Build Cave levels, implementing hazards (darkness, puzzles).  
    * Implement Cave enemies and Final Boss.  
    * Integrate Cave music and SFX.  
  * **Ongoing:** Refine HUD, Menus, implement Save/Load system, optimize performance.  
* **Milestones:**  
  * Environment Complete (Forest, Desert, Mountain, Cave) \- Levels, assets, mechanics integrated.  
  * Alpha Build: All levels and core mechanics implemented, playable start to finish (bugs expected).

### **Phase 4: Post-Production (8-12 Weeks)**

* **Goal:** Polish, bug fix, optimize, prepare for release.  
* **Tasks:**  
  * **Beta Testing:** Gather feedback from testers, identify bugs and usability issues.  
  * **Bug Fixing:** Prioritize and fix critical bugs.  
  * **Polishing:** Refine animations, visual effects, sound mixing, level flow, difficulty balancing.  
  * **Optimization:** Ensure stable 60 FPS on target platforms (PC, potentially mobile profiling).  
  * **Platform Integration:** Implement platform-specific requirements (e.g., Steam integration, mobile controls/builds).  
  * **Localization (if applicable):** Prepare text for translation.  
  * Create marketing materials (trailer, screenshots).  
* **Milestones:**  
  * Beta Build: Feature complete, relatively stable, ready for wider testing.  
  * Release Candidate: Final build, rigorously tested, ready for submission/release.

### **Phase 5: Launch & Post-Launch (Ongoing)**

* **Goal:** Release the game and provide support.  
* **Tasks:**  
  * Launch on target platforms (PC).  
  * Monitor player feedback and reviews.  
  * Release patches for critical bugs.  
  * Consider updates or DLC (if planned).  
  * Mobile port development (if pursued post-PC launch).

## **4\. Estimated Total Timeline**

* **Core PC Development:** Approximately 44 \- 66 weeks (10 \- 15 months).  
* *Note: This is highly variable. A very experienced team might be faster; unforeseen issues can cause delays.*

## **5\. Key Dependencies & Risks**

* **Team Availability & Skillset:** Ensuring team members have the required skills (Unity C\#, pixel art, level design, sound).  
* **Scope Creep:** Adding features not initially planned can significantly delay the project. Stick to the GDD.  
* **Technical Challenges:** Performance optimization (especially for mobile), complex boss logic, pixel-perfect rendering consistency.  
* **Asset Creation Bottleneck:** Pixel art and animation can be time-consuming. Ensure realistic estimates for asset delivery.  
* **Playtesting Feedback:** Major design changes required based on feedback can impact schedule.  
* **Burnout:** Maintaining motivation and avoiding burnout on a long project.

This plan provides a basic framework. Regular reviews, adjustments based on progress, and clear communication within the team will be crucial for success.
